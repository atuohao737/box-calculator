# 📦 标准木箱装箱计算器 — 阶段总结

> **版本**: v1.0  
> **日期**: 2025-06-05  
> **文件**: `index.html` (单文件应用, ~2577行)  
> **技术栈**: 纯前端 (HTML + CSS + JavaScript + Three.js)

---

## 一、项目概述

一个专业的木箱装箱计算工具。用户输入木箱内径和纸箱尺寸后，程序自动计算最优排列方案，并以 3D 可视化方式展示。支持**单品对比**和**混装**两种计算模式。

---

## 二、架构设计

### 模块划分（6个模块，纯 IIFE 模式，无框架依赖）

| 模块 | 职责 |
|------|------|
| `AppState` | 全局状态管理 — 纸箱列表、计算模式、方案索引、优化策略 |
| `PackingEngine` | 装箱算法 — 网格层叠算法 + 贪心空间分割算法 + 智能组合探索器 |
| `StorageManager` | 本地存储 — localStorage 管理规格库和历史记录 |
| `Visualizer3D` | 3D 可视化 — Three.js 场景渲染、相机控制、hover 检测 |
| `UIRenderer` | UI 渲染 — 动态生成纸箱列表、结果卡片、方案标签、历史记录 |
| `App` | 主控制器 — 事件绑定、模块协调、用户交互处理 |

### 设计模式

- **模块化 IIFE**：每个模块独立作用域，通过闭包暴露公共 API
- **状态集中管理**：`AppState` 作为单一数据源，其他模块读取但不直接修改
- **发布-订阅**：模块间通过 `App` 主控制器协调，避免直接耦合

## 三、核心功能

### 3.1 两种装箱算法

#### A. 单品网格层叠算法 (`calcPacking`)
- 尝试 6 种旋转方向（长宽高排列组合）
- 三轴整数排列 + Z 轴层叠
- X/Y 轴尾余填充（利用剩余空间放旋转纸箱）
- 择优：数量多优先 → 利用率高优先
- 支持 `keepUpright` 限制（禁止竖放）

#### B. 混装贪心空间分割算法 (`calcMixedPacking`)
- 贪心 + 空间分割（三维 X/Y/Z 方向）
- 最小浪费优先选择空间
- 多重试机制（Fisher-Yates 洗牌打乱顺序，最多50次）
- 两种策略：数量优先 / 空间优先
- 智能组合探索器（2-3种无数量限制纸箱时触发）
- 需求数量支持（留空 = 尽量多放）

### 3.2 3D 可视化

- **Three.js** WebGL 渲染，支持阴影和抗锯齿
- **木箱模型**：半透明褐色 BoxGeometry + 金色线框
- **纸箱模型**：不透明彩色 BoxGeometry + 白色线框 + 顶面方向标
- **方向标**：悬浮在原始顶面的黑白圆形标识，旋转过的纸箱方向标跟随旋转
- **性能优化**：InstancedMesh 批量渲染相同尺寸纸箱
- **交互**：鼠标旋转/平移/缩放、触摸手势、Hover 信息提示
- **控制**：重置视角、线框模式、木箱显隐、方向标显隐

### 3.3 数据管理

- **纸箱规格库**：保存/加载/删除/导入常用纸箱尺寸
- **木箱尺寸库**：保存/加载/删除/导入常用木箱尺寸
- **历史记录**：自动保存每次计算结果（最多50条），支持回看和删除
- **导出功能**：计算结果导出为 .txt 文件

### 3.4 视觉设计

- **颜色方案**：8 种预设颜色，旋转过的纸箱同色系稍暗
- **方向标**：悬浮于顶面，白色底层 + 黑色实心圆 + 环形描边
- **纸箱不透明度**：0.95（近乎不透明，清晰可见内部排列）
- **响应式布局**：桌面双栏 / 移动端单栏 + 滑入抽屉 + 底部 Tab 导航

## 七、后续可扩展方向

### 1. 更多装箱算法

当前单品模式使用**网格层叠算法**，混装模式使用**贪心空间分割算法**。两者都属于"构造型启发式"，速度快但可能不是全局最优解。可引入以下算法：

#### 1.1 模拟退火 (Simulated Annealing)
- **原理**：模拟金属退火过程，从一个初始解开始，随机扰动（如交换两个纸箱位置、改变某个纸箱的旋转方向），如果新解更好则接受，如果更差则以一定概率接受（概率随"温度"下降而降低）
- **优势**：能跳出局部最优，找到接近全局最优的解
- **代价**：计算时间较长，需要调参（初始温度、降温速率、终止温度）
- **适用场景**：对利用率要求极高的场景

#### 1.2 遗传算法 (Genetic Algorithm)
- **原理**：维护一个解"种群"，每个解是一个装箱方案。通过交叉（两个方案交换部分排列）、变异（随机改变某个纸箱的旋转/位置）、选择（保留利用率高的方案）迭代进化
- **优势**：天然支持并行，适合大规模问题
- **代价**：编码方案设计复杂，需要调参（种群大小、交叉率、变异率、迭代代数）

#### 1.3 分支定界 (Branch and Bound)
- **原理**：系统搜索所有可能排列，但通过"上界估计"剪枝——如果某个分支即使最优情况也不如当前已知最优解，则跳过
- **优势**：可证明找到的是全局最优解
- **代价**：纸箱数量多时计算量爆炸，需要配合启发式剪枝
- **适用场景**：纸箱种类少、精度要求极高的场景

#### 1.4 实现建议
在计算选项中增加"算法选择"下拉框，用户可在"快速"（当前算法）、"精确"（分支定界，小规模）、"智能"（模拟退火/遗传，大规模）之间切换。

### 2. 3D 动画

当前 3D 视图是静态的——计算完成后直接显示最终排列。加入动画可以让用户直观理解装箱过程。

#### 2.1 逐层/逐个装入动画
- **逐层展示**：从底部开始，一层一层地"升起"纸箱，每层间隔 0.3-0.5 秒
- **逐个装入**：按算法放置顺序，纸箱从木箱上方"掉落"到目标位置
- **实现方式**：使用 `gsap` (GreenSock) 或 Three.js 自带的 `requestAnimationFrame` 做位置/透明度插值

#### 2.2 旋转动画
- 纸箱从原始朝向平滑旋转到最终朝向
- 方向标随纸箱旋转，清晰展示"这个纸箱翻了个面"

#### 2.3 阶段高亮
- 先显示主排列区域（高亮），再显示尾余填充区域（另一种颜色高亮）
- 混装模式：每种纸箱依次装入，当前装入的纸箱闪烁或发光

#### 2.4 时间线控制
- 底部增加播放/暂停/快进/快退按钮
- 进度条显示当前已装入百分比

#### 2.5 技术要点
- 需要将最终位置作为"目标"，创建过渡动画
- 大量纸箱时需要优化（超过 200 个可关闭动画，切回静态模式）
- 可用 `THREE.AnimationMixer` 或自己写简单的补间系统

### 3. 导出格式扩展

当前只支持导出 `.txt` 纯文本。可以扩展以下格式：

#### 3.1 PDF 报告
- **内容**：木箱信息、纸箱清单、计算结果（数量、利用率、排列方式）、3D 视图截图
- **实现**：使用 `html2canvas` 截取 3D 画布 + `jspdf` 生成 PDF
- **附加**：可加公司 Logo、日期、计算人签名栏

#### 3.2 图片导出
- **PNG 高清截图**：使用 `renderer.domElement.toDataURL()` 获取 3D 画布的截图
- **多角度导出**：自动旋转到正视图/俯视图/侧视图各拍一张
- **标注**：在图片上叠加尺寸标注线

#### 3.3 CSV/Excel 导出
- **装箱清单**：每行一个纸箱，列出序号、名称、位置坐标、旋转状态
- **汇总表**：各规格纸箱的数量、体积汇总
- **实现**：纯前端用 `xlsx` 库生成 `.xlsx` 文件

#### 3.4 可打印装箱指导图
- 分层打印：每层纸箱的俯视图排列，标注位置编号
- 工人可按照图纸逐层装箱

### 5. 批量计算

当前一次只能计算一种木箱尺寸。批量计算可以一次输入多种木箱，自动找到最佳匹配。

#### 5.1 使用场景
- 工厂有 3 种标准木箱尺寸（小/中/大），想知道这批纸箱用哪种木箱最合适
- 客户给出纸箱清单，需要报价，自动计算用哪种木箱成本最低

#### 5.2 功能设计
1. **输入多种木箱**：木箱尺寸区域支持添加多个（如 1200×1000×800、1100×1100×900...）
2. **逐个计算**：对每种木箱分别运行当前算法
3. **结果对比表**：

| 木箱尺寸 | 装箱数量 | 利用率 | 剩余空间 | 推荐 |
|---------|---------|--------|---------|------|
| 1200×1000×800 | 48个 | 78.5% | 0.21m³ | ⭐ |
| 1100×1100×900 | 52个 | 82.1% | 0.18m³ | ⭐⭐ 最佳 |
| 1000×800×700  | 30个 | 91.2% | 0.09m³ | |

4. **成本估算**：如果木箱有单价，可计算"单件纸箱的木箱成本"
5. **推荐逻辑**：利用率 × 数量 × 成本，综合排序

### 6. 约束条件扩展

当前只考虑了几何尺寸约束。实际装箱场景还有更多限制：

#### 6.1 重量限制
- **纸箱重量**：每种纸箱可输入单件重量
- **木箱承重**：木箱有最大承重限制
- **层间承重**：下层纸箱是否能承受上层重量（可设置每层最大承重）
- **算法影响**：重的纸箱优先放底层，超出承重则减少该层数量

#### 6.2 重心计算
- **目标**：装箱后整体重心尽量靠近木箱几何中心，避免运输时倾斜
- **计算方式**：`重心 = Σ(纸箱位置 × 纸箱重量) / 总重量`
- **显示**：3D 视图中用小球或十字线标示重心位置
- **警告**：重心偏移超过阈值（如 15%）时给出提示

#### 6.3 堆叠强度
- 纸箱有不同的抗压强度等级
- 底层纸箱需要承受上方所有纸箱的重量
- 如果底层纸箱抗压不够，需要调整排列或减少层数

#### 6.4 特殊约束
- **标签面朝外**：某些纸箱要求标签面必须朝向木箱外侧
- **不可倒置**：箭头向上的纸箱不能倒放（已有 `keepUpright` 功能，可扩展为更细粒度的朝向限制）
- **危险品隔离**：某些纸箱不能相邻放置
- **温度敏感**：某些纸箱不能靠近木箱边缘（温差大）

### 7. PWA 支持

PWA (Progressive Web App) 可以让这个工具像原生 App 一样使用。

#### 7.1 离线使用
- **Service Worker**：缓存 `index.html` 和 Three.js CDN 资源
- 首次访问后，即使没有网络也能打开使用
- Three.js 需要缓存到本地（或改用 npm 本地版本）

#### 7.2 安装到桌面
- **Web App Manifest**：配置 `manifest.json`，定义图标、名称、启动方式
- 用户在浏览器中点击"添加到主屏幕"，桌面出现应用图标
- 启动时无浏览器地址栏，全屏体验

#### 7.3 推送通知（远期）
- 计算结果完成时推送通知（如果计算迁移到后端）
- 规格库更新提醒

#### 7.4 实现步骤
1. 添加 `manifest.json`（配置图标、名称、主题色）
2. 注册 Service Worker（缓存策略：优先缓存，网络更新）
3. 添加安装提示按钮（当浏览器检测到 PWA 可安装时显示）
4. 生成多尺寸图标（192×192、512×512 等）

#### 7.5 优势
- 工厂车间工人手机上一键打开，无需输入网址
- 离线也能用，不受车间网络影响
- 体验接近原生 App，但无需应用商店审核
