# 同层混向网格算法（Mixed-Orientation Grid）— 实施方案

## 算法命名

**同层混向网格**（英文：Mixed-Orientation Grid / MOG）

在同一层内，允许纸箱以不同的方向混合排列，而不是全部朝向一致。

## 问题现状

对于 1020×1020×930 木箱 + 360×280×210 纸箱：

| 方案 | 计算方式 | 每层数量 | 总数量 |
|------|----------|---------|--------|
| 纯横放 | floor(1020/360)=2 × floor(1020/280)=3 | 6 | 24 |
| 纯竖放 | floor(1020/280)=3 × floor(1020/360)=2 | 6 | 24 |
| 师傅方案 | 混合方向 — 280+280+210+210 ↑↓ 方向 | 16 | ~41 |
| **差距** | **混合排列 vs 单一方向** | **+167%** | **+71%** |

## 算法设计

### 核心变化

在 `calcPacking` 的主网格层叠环节（第 50-57 行），增加一个**二维混合网格枚举**步骤，替代原有的单一方向 `xCount × yCount` 循环。

### 混合网格枚举逻辑

对于给定的纸箱尺寸 `(bL, bW, bH)` 和木箱可用空间 `(cL, cW)`：

```
函数 calcMixedGrid(cL, cW, bL, bW):
    枚举: 横向占位宽 = bL, 纵向占位宽 = bW
          竖向占位宽 = bW, 纵向占位宽 = bL
    即：横放占用 (bL, bW)，竖放占用 (bW, bL)

    遍历横向行数 hRows (0 到 floor(cL / bL)):
        遍历横向列数 hCols (0 到 floor(cW / bW)):
            遍历竖向行数 vRows (0 到 floor(cL / bW)):
                遍历竖向列数 vCols (0 到 floor(cW / bL)):
                    检查横放区域: hRows × bL + vRows × bW ≤ cL
                    检查竖放区域: hCols × bW + vCols × bL ≤ cW  
                    （其实就是看两排横放后能不能再塞一排竖放）
        
    找最大格子数 hRows×hCols + vRows×vCols
```

### 更高效的枚举方式

直接 4 层循环太慢。更好的方法：

```
对于每种旋转方向 (bL, bW):
    枚举横放行数 hx（0 到 floor(cL / bL))
    枚举竖放行数 vx（0 到 floor(cL / bW))
    检查 bL × hx + bW × vx ≤ cL
    
    计算每行能放的数量:
        横放每行: floor(cW / bW)
        竖放每行: floor((cW - hxUsed) / bL)
    
    总数量 = hx × floor(cW / bW) + vx × floor((cW - UsedX) / bL)
    取最大值
```

相当于只在 X 方向混排（不同行/列用不同朝向），Y 方向纯用。

### 更优的通用方案：枚举分界点

对于二维矩形装箱问题，最优解可以通过枚举**分界点**来找到：

1. 沿长边（X轴），枚举分界点 `splitX`，0 ≤ splitX ≤ cL
2. 左边区域 `[0, splitX]` 放横放箱子（朝向 bL×bW）
3. 右边区域 `[splitX, cL]` 放竖放箱子（朝向 bW×bL）
4. 对每个分界点，计算左右两边各自能放多少

分界点只需要枚举箱子尺寸的整数倍：splitX = n × bL 或 splitX = n × bW

```
最佳方案 = 0
对于 hCount = 0 到 floor(cL / bL):
    usedByH = hCount × bL
    横放总格子 = hCount × floor(cW / bW)
    左侧剩余 = cL - usedByH
    竖放行数 = floor(左侧剩余 / bW)
    竖放列数 = floor(cW / bL)
    竖放总格子 = 竖放行数 × 竖放列数
    总数 = 横放总格子 + 竖放总格子
    如果总数 > 最佳方案，更新

对称再算一遍（先竖后横）
```

## 3. 算法设计

### 3.1 二维分界枚举法

在 `calcPacking` 的主网格计算环节，对每个旋转方向 `(bL, bW, bH)` 增加二维分界枚举：

```
对于给定的层高 bH 和平面尺寸 (cL, cW):

枚举 X 方向的分界 splitX:
  splitX 来自 bL 的整数倍 (横放列数 xh × bL) 
  或者 bW 的整数倍 (竖放列数 xv × bW)
  
  X 方向排列 = xh 列横放 + xv 列竖放
  约束: xh × bL + xv × bW ≤ cL

  在占据的 X 空间内，枚举 Y 方向的排列:
  枚举 yh (横放行数), yv (竖放行数)
  约束: yh × bW + yv × bL ≤ cW
  
  总格子数 = xh × yh + xv × yv + 
            (xv × yh + xh × yv) 
  
  注意每个位置的朝向由行列共同决定:
  - (横放列, 横放行) → 正常放置 (bL×bW)
  - (竖放列, 竖放行) → 旋转后放置 (bL×bW)
    实际上竖放列+竖放行 = (bW×bL) 旋转90度仍为 (bW×bL)?
  
  简单做法：每个位置固定为 (bL, bW) 或 (bW, bL)
  - 横放列: 该列占用 bL 宽度，格子宽 bL
  - 竖放列: 该列占用 bW 宽度，格子宽 bW
  - 横放行: 该行占用 bW 高度，格子高 bW
  - 竖放行: 该行占用 bL 高度，格子高 bL
  
  所以格子朝向 = (列宽, 行高)
  - 横放列 + 横放行 = (bL, bW) → 横放
  - 横放列 + 竖放行 = (bL, bL) → 可能不合法
```

### 3.2 简化的实现方式

**一行枚举法**（简化版）：

只沿 X 方向做分界，Y 方向用同一种朝向填充：

```
枚举横放列数 xh (0..floor(cL/bL)):
  枚举竖放列数 xv (0..floor(cL/bW)):
    usedX = xh×bL + xv×bW
    if usedX > cL: continue
    
    // 横放区域每行的格子数
    yPerRowH = floor(cW / bW)
    // 竖放区域每行的格子数
    yPerRowV = floor(cW / bL)
    
    但横竖区域行高不同:
    横放区行高 = bW, 竖放区行高 = bL
    不能简单合并...
    
    所以需要回退到二维枚举
```

### 3.3 推荐的最终实现

由于 bL 和 bW 通常在几百毫米量级，cL 和 cW 在 1000mm 量级，
行列数最多 = max(floor(1000/200), floor(1000/400)) ≈ 5

所以枚举非常快，可以使用最简单的四层循环：

```
函数 calcMixedGrid(cL, cW, bL, bW):
    bestCount = 0
    bestConfig = null
    
    // 枚举横放列数
    for xh = 0 to floor(cL / bL):
        // 枚举竖放列数 
        for xv = 0 to floor(cL / bW):
            usedX = xh×bL + xv×bW
            if usedX > cL: continue
            
            // 枚举横放行数
            for yh = 0 to floor(cW / bW):
                // 枚举竖放行数
                for yv = 0 to floor(cW / bL):
                    usedY = yh×bW + yv×bL
                    if usedY > cW: continue
                    
                    total = xh×yh + xv×yv + xh×yv + xv×yh
                    // = (xh + xv)(yh + yv)
                    
                    if total > bestCount:
                        bestCount = total
                        bestConfig = {xh, xv, yh, yv}
    
    return bestConfig, bestCount
```

这个枚举的复杂度是 O(k²j²)，其中 k ≤ floor(cL/min(bL,bW)) ≤ 10，j 同理。最多 10⁴ = 10000 次迭代，极快。

### 3.4 位置生成

找到最优的 (xh, xv, yh, yv) 后，生成具体的坐标：

```
for i = 0 to xh-1:  // 横放列
    for j = 0 to yh-1:  // 横放行
        位置: (gap + i×bL, gap + j×bW, z) 尺寸: (bL, bW, bH)
    for j = 0 to yv-1:  // 竖放行
        位置: (gap + i×bL, gap + yh×bW + j×bL, z) 尺寸: (bL, bW, bH)
        这个位置行高是 bL，但列宽是 bL，所以实际上箱子的朝向...
```

这里我意识到一个问题：**当横放列 + 竖放行时，格子的宽 = bL（列）、高 = bL（行），形成一个 bL×bL 的方格，但箱子是 bL×bW，不一定放得进去。**

所以不是所有交叉组合都有效，需要额外检查：
- (横放列, 横放行): 箱子占用 bL×bW，格子 bL×bW ✓
- (横放列, 竖放行): 箱子占用 bL×bW，格子 bL×bL ✗（除非 bL=bW）
- (竖放列, 横放行): 箱子占用 bW×bL，格子 bW×bW ✗
- (竖放列, 竖放行): 箱子占用 bW×bL，格子 bW×bL ✓

### 3.6 最终采用的算法：沿单轴分界枚举

```
函数 calcMixedGrid(cL, cW, bL, bW):
    bestTotal = 0, bestConfig = null
    
    // 方案A：X方向先横放，剩余放竖放
    for hx = 0 to floor(cL / bL):
        usedX = hx × bL
        remX = cL - usedX
        vx = floor(remX / bW)
        
        // 横放部分:
        //   X方向 hx 个位置，Y方向 floor(cW / bW) 个位置
        hTotal = hx × floor(cW / bW)
        
        // 竖放部分:
        //   X方向 vx 个位置，Y方向 floor(cW / bL) 个位置
        vTotal = vx × floor(cW / bL)
        
        total = hTotal + vTotal
        if total > bestTotal:
            bestTotal = total; bestConfig = {hx, vx, type: 'X-split'}
    
    // 方案B：Y方向先横放，剩余放竖放
    for hy = 0 to floor(cW / bW):
        usedY = hy × bW
        remY = cW - usedY
        vy = floor(remY / bL)
        
        hTotal = hy × floor(cL / bL)
        vTotal = vy × floor(cL / bW)
        total = hTotal + vTotal
        if total > bestTotal:
            bestTotal = total; bestConfig = {hy, vy, type: 'Y-split'}
    
    // 方案C：X方向先竖放，剩余放横放
    for vx = 0 to floor(cL / bW):
        usedX = vx × bW
        remX = cL - usedX
        hx = floor(remX / bL)
        
        vTotal = vx × floor(cW / bL)
        hTotal = hx × floor(cW / bW)
        total = hTotal + vTotal
        if total > bestTotal:
            bestTotal = total; bestConfig = {vx, hx, type: 'X-split-rev'}
    
    // 方案D：Y方向先竖放，剩余放横放
    for vy = 0 to floor(cW / bL):
        usedY = vy × bL
        remY = cW - usedY
        hy = floor(remY / bW)
        
        vTotal = vy × floor(cL / bW)
        hTotal = hy × floor(cL / bL)
        total = hTotal + vTotal
        if total > bestTotal:
            bestTotal = total; bestConfig = {vy, hy, type: 'Y-split-rev'}
    
    return bestTotal, bestConfig
```

## 算法名称

**同层混向网格** — Mixed-Orientation Grid (MOG)

在同一水平层内，纸箱可以混合两种朝向（横放/竖放），而不是全部朝向一致。

## 问题

当前 `calcPacking` 的网格层叠对每种旋转方向计算纯网格：
```
xCount = floor(cL / bL), yCount = floor(cW / bW)
```

同一层内所有箱子朝向一致，剩余空间只能靠尾余填充。

**师傅案例（1020×1020×930 + 360×280×210）：**

| 方案 | 每层 | 总数量 |
|------|------|--------|
| 纯横放 (360朝X) | 6 | 24 |
| 纯竖放 (280朝X) | 6 | 24 |
| 师傅方案（混排） | **16** | **~41** |
| 差异 | **+167%** | **+71%** |

## 改动位置

`packing-engine.js` — `calcPacking` 函数，主网格层叠环节（当前第 50-57 行）

替换原有的纯网格生成，改为**沿单轴分界枚举**——在 X 或 Y 方向设分界点，一边横放一边竖放。

## 算法描述

### 核心思路

```
cL × cW 的矩形平面内：
┌──────────────┬──────────┐
│              │          │
│   横放区域    │  竖放区域 │
│   (bL×bW)    │  (bW×bL) │
│              │          │
└──────────────┴──────────┘
     ← 分界点 →
```

沿 X 方向枚举分界点，左边区域放横放箱子（bL朝X），右边区域放竖放箱子（bW朝X）。

### 枚举逻辑

```
函数 calcMixedGrid(cL, cW, bL, bW):
    best = 0, bestConfig = null
    
    // 枚举 X 方向分界: hx 列横放, vx 列竖放
    for hx = 0..floor(cL/bL):
        remX = cL - hx × bL
        vx = floor(remX / bW)
        
        // 横放区 Y 方向格子数
        hy_count = floor(cW / bW)
        // 竖放区 Y 方向格子数(Y方向用bL约束)
        vy_count = floor(cW / bL)
        
        total = hx × hy_count + vx × vy_count
        if total > best: best = total; config = {hx, vx, dir: 'X'}
    
    // 对称: Y 方向分界
    for hy = 0..floor(cW/bW):
        remY = cW - hy × bW
        vy = floor(remY / bL)
        
        hx_count = floor(cL / bL)
        vx_count = floor(cL / bW)
        
        total = hy × hx_count + vy × vx_count
        if total > best: best = total; config = {hy, vy, dir: 'Y'}
    
    return best, config
```

### 位置生成

```
如果最优配置是 X 方向分界：
  // 横放区域
  for x = 0..hx-1, y = 0..hy_count-1:
    位置 (gap + x×bL, gap + y×bW, z)  尺寸 (bL, bW, bH)
  
  // 竖放区域
  for x = 0..vx-1, y = 0..vy_count-1:
    位置 (gap + hx×bL + x×bW, gap + y×bL, z)  尺寸 (bW, bL, bH)
```

### 与现有 X/Y 尾余填充的关系

MOG 替换的是**主网格**（第 50-57 行的三循环），X/Y 尾余填充和 Z 顶层填充保持不变，继续在 MOG 之后执行。

这样：
1. 主网格用 MOG 混合朝向 → 比纯网格多装
2. X 尾余填充 MOG 区域在 X 方向剩余 → 如果还有剩装更多
3. Y 尾余填充 MOG 区域在 Y 方向剩余 → 同上
4. Z 顶层填充顶部空间 → 同上
