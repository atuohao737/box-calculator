## 🔴 P0 — 立即修复

### P0-1: `calcEnumPackingOptimized` 忽略 gap 参数

**文件**: `src/js/optimized-packing.js`  
**行号**: 128~154, 206~229  
**类型**: 正确性 bug

**问题**: 函数接收 `gap` 参数但从未使用。原始 `calcEnumPacking`（packing-engine.js:27）入扣减 `cL = crateL - gap * 2`，坐标加 `gap + c.x` 偏移。优化版全程使用原始 crate 尺寸，坐标从 (0,0,0) 开始。

**影响**: `gap > 0` 时优化算法会过度装满、坐标元数据错误，3D视图展示位置偏移。

**修复步骤**:

1. 在 `calcEnumPackingOptimized` 入口扣减 gap：
   ```javascript
   const iL = cL - gap * 2, iW = cW - gap * 2, iH = cH - gap * 2;
   if (iL <= 0 || iW <= 0 || iH <= 0) return null;
   ```
2. 将 `iL, iW, iH` 传给 `onePass` 和内部函数
3. 在 `positions` 生成处（L212, L224）加 gap 偏移：
   ```javascript
   x: gap + c.x, y: gap + c.y, z: gap + zOff
   ```

**预计改动量**: ~15 行

### P0-2: `calcEnumPackingOptimized` 忽略 `keepUpright` 约束

**文件**: `src/js/optimized-packing.js` (L128~130) + `src/js/packing-engine.js` (L430~431)  
**类型**: 正确性 bug

**问题**: 优化版 `getRotations()` 硬编码 `keepUpright: false`，调用侧未传入。用户勾选"禁止倒置"后在优化路径被忽略。

**修复步骤**:

1. `optimized-packing.js` L128 增加 `keepUpright` 参数：
   ```javascript
   function calcEnumPackingOptimized(cL, cW, cH, bL, bW, bH, gap, allowRotate, keepUpright, timeLimitMs)
   ```
2. L130 改用参数：
   ```javascript
   const rots = getRotations(bL, bW, bH, allowRotate, keepUpright);
   ```
3. `packing-engine.js` L431 传入 `keepUpright`：
   ```javascript
   var enumR2 = PE.calcEnumPackingOptimized(crateL, crateW, crateH, boxL, boxW, boxH, gap, allowRotate, keepUpright, enumTimeLimit);
   ```

**预计改动量**: ~5 行

## 🟡 P1 — 本迭代修复

### P1-1: `onePass` 超时控制翻倍 bug

**文件**: `src/js/optimized-packing.js` L163~166  
**类型**: 性能边界

**问题**: `onePass` 的 `timeLimitMs` 实参是 `perPass`（= total/3），但循环中超时比较用了 `*2`，导致实际运行上限是声明的 ~2 倍。

**修复**: L165 改为 `if (Date.now() - globalT0 > timeLimitMs) break;` 或按 `perRotBudget` 累加。

**预计改动量**: 2 行

---

### P1-2: `maxByVol` 使用原始尺寸导致过早退出

**文件**: `src/js/optimized-packing.js` L141  
**类型**: 算法效率

**问题**: 理论最大箱数按原始尺寸计算，当前 pass 使用旋转后朝向时，实际最大值可能不同，导致提前退出错过更优解。

**修复**: 在 pass 内获取当前最优朝向的实际尺寸来计算 maxByVol。

**预计改动量**: 3 行

---

### P1-3: 测试未加载优化模块

**文件**: `tests/setup.js` L11~17  
**类型**: 测试覆盖

**问题**: `moduleFiles` 缺少 `optimized-packing.js`，所有测试跳过优化路径。packing-engine.js L430 的 `typeof calcEnumPackingOptimized` 永远为 false。

**修复**: 在 `app-state.js` 和 `packing-engine.js` 之间插入：
```javascript

```
确保加载顺序：`app-state.js → packing-engine.js → optimized-packing.js → storage-manager.js → ...`

**预计改动量**: 1 行
