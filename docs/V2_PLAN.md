# 📦 标准木箱装箱计算器 V2 规划文档

> **日期**: 2026-06-05  
> **状态**: 规划阶段

---

## V2 核心目标

从"给定木箱，算怎么装"升级为"给定纸箱，告诉我用什么木箱、怎么装、装不装得下"。

---

## 一、批量计算

### 1.1 背景

V1 一次只能计算一种木箱尺寸。实际场景中，工厂通常有多种标准木箱，需要快速对比"这批纸箱用哪种木箱最合适"。

### 1.2 功能描述

#### 输入层
- 木箱尺寸区域支持**添加多个**木箱规格
- 可从木箱规格库**批量导入**
- 每个木箱可设置独立参数（如承重、单价）
- 支持手动添加临时木箱（不存入规格库）

#### 计算层
- 对每种木箱分别运行 V1 现有算法
- 可并行计算（Web Worker），避免界面卡顿
- 显示每种木箱的计算进度

#### 结果层 — 对比表

| 木箱尺寸 | 装箱数量 | 利用率 | 剩余空间 | 推荐 |
|---------|---------|--------|---------|------|
| 1200×1000×800 | 48个 | 78.5% | 0.21m³ | ⭐ |
| 1100×1100×900 | 52个 | 82.1% | 0.18m³ | ⭐⭐ 最佳 |
| 1000×800×700  | 30个 | 91.2% | 0.09m³ | |

#### 推荐逻辑
- **默认推荐**：利用率最高
- **成本推荐**（如果填写了单价）：单件纸箱的木箱成本最低
- **数量推荐**：装得最多的
- 用户可在设置中切换推荐策略

#### 3D 展示
- 对比表中点击某行，3D 视图切换为该木箱的装箱方案
- 支持并排对比模式（分屏显示两种方案的 3D 视图）

### 1.3 技术要点

- 计算量 = 木箱数 × 算法复杂度，需要进度反馈
- 如果木箱数量 > 2 且纸箱数量多，考虑用 Web Worker 避免 UI 阻塞
- 对比表需要支持排序（按利用率/数量/成本）

## 二、木箱反推

### 2.1 背景

V1 的流程是：用户已知木箱尺寸 → 计算怎么装。但很多场景是反过来：
- 客户给了一份纸箱清单，问"要几个木箱？多大？"
- 仓库有标准木箱尺寸，想知道用哪种、用几个

### 2.2 功能描述

#### 输入
- 纸箱清单（和 V1 一样：尺寸、数量、名称、是否保持竖放）
- 木箱标准尺寸库（用户预设的常用木箱尺寸）

#### 计算流程

```
纸箱清单
   ↓
按木箱尺寸从小到大尝试
   ↓
对每种木箱尺寸，用贪心算法逐箱填装：
  1. 从纸箱列表中取出未分配的纸箱
  2. 装入当前木箱（用 V1 混装算法）
  3. 如果当前木箱装不下更多，开新木箱
  4. 重复直到所有纸箱分配完毕
   ↓
输出：每种木箱尺寸需要的数量
   ↓
对比推荐最优方案
```

#### 输出结果

**方案概览**：

| 木箱尺寸 | 需要数量 | 总利用率 | 单箱利用率范围 | 推荐 |
|---------|---------|---------|--------------|------|
| 1200×1000×800 | 3个 | 81.2% | 76%-88% | ⭐⭐ 最佳 |
| 1100×1100×900 | 4个 | 73.5% | 65%-82% | |
| 1500×1200×1000 | 2个 | 68.1% | 62%-74% | |

**每个木箱详情**（展开某行后）：
- 木箱 #1：装了 12 个 A + 8 个 B，利用率 88%
- 木箱 #2：装了 10 个 A + 10 个 B，利用率 82%
- 木箱 #3：装了 8 个 A + 4 个 B，利用率 76%

#### 推荐逻辑
- **数量最少优先**：总木箱数最少
- **利用率最高优先**：总利用率最高
- **成本最低优先**（如果有单价）

#### 3D 展示
- 点击某个木箱查看其 3D 装箱方案
- 支持逐个木箱浏览（上一个/下一个）

### 2.3 边界情况

| 情况 | 处理 |
|------|------|
| 某纸箱尺寸超过所有木箱 | 标记为"不可装"，提示用户 |
| 纸箱总数非常少 | 可能只需 1 个木箱，退化为 V1 单品/混装 |
| 标准木箱库为空 | 提示用户先添加木箱尺寸 |
| 某种纸箱数量 = 0 | 跳过该规格 |
| 计算结果差距大 | 高亮显示利用率异常低/高的木箱 |

### 2.4 技术要点

- 核心是**多箱装箱问题 (Multiple Bin Packing)**，比 V1 的单箱装箱更复杂
- 策略：First Fit Decreasing (FFD) — 先装大纸箱，每个木箱尽量装满再开新的
- 可尝试多种排列顺序，取总木箱数最少的结果
- 如果标准木箱尺寸很多（>10种），需要限制尝试次数或给用户"快速/精确"选项

## 三、重量约束

### 3.1 背景

V1 只考虑了几何尺寸。实际装箱中，木箱有承重上限，纸箱有自身重量，底层纸箱还需承受上层重量。

### 3.2 数据模型扩展

#### 纸箱新增字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 单件重量 | number (kg) | 每个纸箱的重量 |
| 抗压强度 | number (kg) | 可选，纸箱能承受的最大压力 |

#### 木箱新增字段
| 字段 | 类型 | 说明 |
|------|------|------|
| 最大承重 | number (kg) | 木箱总承重上限 |

### 3.3 约束规则

#### 3.3.1 总重限制
- 木箱内所有纸箱重量之和 ≤ 木箱最大承重
- 超出时：减少纸箱数量 → 提示用户"承重已满，建议分箱"

#### 3.3.2 底层承重（可选，如果填写了抗压强度）
- 底层每个纸箱承受的压力 = 上层纸箱总重量 / 底层纸箱接触面积
- 超出抗压强度时 → 调整排列（减少层数 / 重的放底层）

#### 3.3.3 重量排序
- 装箱前按重量降序排列纸箱（重的先装 → 自然放在底部）
- 这是启发式规则，不是硬约束，但能大幅提升稳定性

### 3.4 UI 设计

#### 输入区域
```
┌─ 纸箱规格 ──────────────────────────┐
│ 名称  长  宽  高  数量  重量(kg)  竖放 │
│ [___] [__][__][__] [__] [____]    ☐  │
│  A    30  20  15  100   2.5       ☐  │
│  B    40  30  20   50   5.0       ☒  │
└──────────────────────────────────────┘

┌─ 木箱参数 ──────────────────────────┐
│ 内径  长 [1200] 宽 [1000] 高 [800]  │
│ 最大承重 [500] kg                    │
└──────────────────────────────────────┘
```

#### 结果展示
- 利用率旁增加"载重率"：`载重 420/500 kg (84%)`
- 如果承重已满但空间有余 → 黄色警告："空间未满但已达承重上限"
- 3D 视图中，纸箱颜色按重量渐变（重的颜色深）

### 3.5 对算法的影响

```
原有装箱流程：
  纸箱列表 → 几何排列 → 输出结果

加入重量后：
  纸箱列表 → 按重量降序排列 → 几何排列 → 检查总重 → 
  ├─ 未超重 → 输出结果
  └─ 超重   → 减少纸箱数量 → 重新排列 → 输出结果 + 警告
```

### 3.6 技术要点

- 重量约束是**后检查**，不改变核心排列算法，改动量较小
- 按重量降序排列是零成本的优化（只改排序，不改算法）
- 承重上限检查在每个木箱计算完成后进行
- 如果 V2 加入反推功能，承重不满时可自动多装其他纸箱

## 四、V2 整体架构变化

```
V1 架构                          V2 架构
─────────                        ─────────
单品模式 (网格层叠)               单品模式 (网格层叠 + 重量检查)
混装模式 (贪心空间分割)           混装模式 (贪心空间分割 + 重量检查)
                                批量计算 (多木箱并行对比)
                                木箱反推 (多箱装箱 FFD)
                                重量约束 (总重/底层承重/重量排序)
```

### 模块变化

| 模块 | V1 | V2 变化 |
|------|-----|--------|
| `AppState` | 单木箱 + 纸箱列表 | + 多木箱列表 + 纸箱重量字段 + 木箱承重字段 |
| `PackingEngine` | 2 种算法 | + 批量计算调度 + 反推算法 + 重量检查 |
| `UIRenderer` | 单结果卡片 | + 对比表 + 反推结果卡片组 + 重量输入 |
| `Visualizer3D` | 单方案展示 | + 方案切换 + 重量渐变着色 |
| `StorageManager` | 规格库/历史 | + 重量数据持久化 |

## 五、开发优先级建议

| 优先级 | 功能 | 理由 |
|--------|------|------|
| 🔴 P0 | 重量约束 | 改动量最小，V1 算法基本不变，见效快 |
| 🟡 P1 | 批量计算 | 多个木箱并行对比，UI 改动较大 |
| 🟢 P2 | 木箱反推 | 新算法，复杂度最高，需要充分测试 |

建议按 P0 → P1 → P2 顺序开发，每个阶段都可独立交付使用。
